Step 1: Dataset Summary & Exploration¶

The pickled data is a dictionary with 4 key/value pairs:

  • 'features' is a 4D array containing raw pixel data of the traffic sign images, (num examples, width, height, channels).
  • 'labels' is a 2D array containing the label/class id of the traffic sign. The file signnames.csv contains id -> name mappings for each id.
  • 'sizes' is a list containing tuples, (width, height) representing the the original width and height the image.
  • 'coords' is a list containing tuples, (x1, y1, x2, y2) representing coordinates of a bounding box around the sign in the image. THESE COORDINATES ASSUME THE ORIGINAL IMAGE. THE PICKLED DATA CONTAINS RESIZED VERSIONS (32 by 32) OF THESE IMAGES

### Implement LeNet-5
from tensorflow.contrib.layers import flatten

# Conv2D wrapper, with bias and relu activation
def conv2d(x, W, b, strides):
    x = tf.nn.conv2d(x, W, strides=[1, strides, strides, 1], padding='VALID')
    x = tf.nn.bias_add(x, b)
    return tf.nn.relu(x)

# Max-pooling
def maxpool2d(x, k):
    return tf.nn.max_pool(x, ksize=[1, k, k, 1], strides=[1, k, k, 1], padding='VALID')

def LeNet(x):        
    # Layer 1
    conv1 = conv2d(x, tf.Variable(tf.truncated_normal(shape=(5, 5, 3, 6), mean = mu, stddev = sigma)), 
                   tf.Variable(tf.zeros(6)), strides)
    conv1 = maxpool2d(conv1, k)

    # Layer 2
    conv2 = conv2d(conv1, tf.Variable(tf.truncated_normal(shape=(5, 5, 6, 16), mean = mu, stddev = sigma)),
                   tf.Variable(tf.zeros(16)), strides)
    conv2 = maxpool2d(conv2, k)
    
    # Fully connected layer
    # Reshape conv3 output to fit fully connected layer input
    fc11 = flatten(conv2)
    fc11 = tf.add(tf.matmul(fc11, tf.Variable(tf.truncated_normal(shape=(400, 120), mean = mu, stddev = sigma))),
                            tf.Variable(tf.zeros(120)))
    fc11 = tf.nn.relu(fc11)
    
    # Fully connected layer
    fc12 = tf.add(tf.matmul(fc11, tf.Variable(tf.truncated_normal(shape=(120, 84), mean = mu, stddev = sigma))),
                            tf.Variable(tf.zeros(84)))
    fc12 = tf.nn.relu(fc12)

    # Output Layer - class prediction
    out = tf.add(tf.matmul(fc12, tf.Variable(tf.truncated_normal(shape=(84, 43), mean = mu, stddev = sigma))),
                           tf.Variable(tf.zeros(43)))
    return out

Question 3¶

What does your final architecture look like? (Type of model, layers, sizes, connectivity, etc.) For reference on how to build a deep neural network using TensorFlow, see Deep Neural Network in TensorFlow from the classroom.

Answer:

Since LeNet inplementation worked well to begin with, I kept its architecture:

  • Convolution layer 1: use (5, 5, 3, 6) filter with 1 stride, the output shape is 28x28x6.
  • Activation 1: use ReLU activation.

  • Pooling layer 1: use max pooling with ksize and stides equal to 2, the output shape is 14x14x6.

  • Convolution layer 2. use (5, 5, 6, 16) filter with 1 stride, the output shape is 10x10x16.

  • Activation 2: use ReLU activation.

  • Pooling layer 2: use max pooling with ksize and stides equal to 2, the output shape is 5x5x16.

  • Flatten layer: flatten the output shape of the final pooling layer such that it's 1D instead of 3D.

  • Fully connected layer 1: it has 120 outputs.

  • Activation 3: use ReLU activation.
  • Fully connected layer 2: it has 84 outputs.

  • Activation 4: use ReLU activation.

  • Fully connected layer 3: this is the output layer. It has 43 outputs.

Question 4¶

How did you train your model? (Type of optimizer, batch size, epochs, hyperparameters, etc.)

Answer:

My testing process is guided by this paper: https://arxiv.org/pdf/1412.6980v8.pdf

  • Optimizer: Adam optimizer

    • Adam optimizer uses moving averages of the parameters, it generally performs better than the Gradient Decent Optimizer. Test confirms this.

  • Learning rate: 0.001

    • Paper suggested Adam optimizer works better with smaller learning rate. I tested few values range(0.001 - 0.01).

  • Batch size: 32

    • Larger batch size yield faster computation but worse performance (since epochs was fixed). Tested 10, 32(default suggested by the paper), 100, 500, 1000 and choose 32.
    • By-batch learning process was plotted and observed to help with the testing

  • EPOCHS: 25

    • Started with 50, algorithm learns very fast after just a few epochs, so 25 is selected.

Since these parameters have implication on each other, several combinations were tested before choosing the final set.

  • Other parameters: mu = 0, sigma = 0.1, strides = 1, k = 2
    • Did not change given the initial setting, achieved high accuracy.
